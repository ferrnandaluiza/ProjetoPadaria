# Projeto Integrador Jovem Programador dez/2021

### Projeto Padock's 

<i>Programa desenvolvido em C# com Visual Studio integrado com banco de dados no Microsoft Access com finalidade de auxiliar gerenciamento completo de uma padaria </i>

#### <li>Visão geral</li>
 <p>O software terá como principal finalidade informatizar alguns processos de uma padaria, com o intuito de otimizar o tempo e custo da produção. Além dos requisitos básicos como os de registrar produtos, colaboradores, clientes e encomendas, contará também com um sistema prático e intuitivo que permitirá com que esteja à disposição dos clientes informações em tempo real acerca dos status de produção dos itens fabricados pela padaria, como quantidade produzida, tempo para finalização do preparo e quantidade disponível para compra. 
  O sistema foi programado para operar em três padrões diferentes: a) o primeiro deles em um totem no local de produção, na cozinha, o qual os colaboradores deverão utilizar para alimentar com as informações a respeito da fabricação dos produtos; b) um segundo totem ou painel com uma tela demonstrativa onde os clientes poderão visualizar as últimas informações a respeito dos produtos disponibilizados para a compra, fornadas previstas e etc; c) ainda, para usuários autorizados, terá uma tela de controle de produtos, pedidos, clientes e funcionários que permita a realização da administração de estoque, de recursos humanos e também financeira, com a função de emissão de recibos fiscais.
Com um sistema integrado garantirá para o cliente sempre transparência da compra de um produto fresco, e, para a padaria, a otimização de sua produção e administração. </p>


#### <li>O software tem os seguintes requisitos funcionais:</li>
1. RF001 O sistema deve permitir o cadastro de um banco de dados com os produtos, colaboradores e clientes da padaria;
2. RF002 O sistema deve permitir que os usuários efetuem login;
3. RF003 O software deve permitir que usuários administradores efetuem a manutenção sobre as informações relacionadas aos colaboradores;
4. RF004 O software deve permitir que usuários administradores efetuem a manutenção sobre as informações relacionadas aos produtos;
5. RF005 O sistema deve disponibilizar uma tela para que usuários administradores atualizem o status de produção dos produtos;
6. RF006 O sistema deve disponibilizar uma tela para que usuários administradores visualizem um resumo da atualização de produção dos produtos;
7. RF007 O sistema deve disponibilizar uma tela para visualizar as encomendas recebidas;
8. RF008 O sistema deve disponibilizar uma tela para que os usuários visualizem os produtos e status de produção do mesmo, como por exemplo: “No forno”, “Preparando a massa” e etc.
